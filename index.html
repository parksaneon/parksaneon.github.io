Props, State

  • props : 컴포넌트 외부에서 컴포넌트에게 주는 데이터

  • state : 컴포넌트 내부에서 변경할 수 있는 데이터

둘다 변경이 발생하면, 렌더가 다시 일어날 수 있다. ( render 함수가 호출돼서 다시 그려진다는 것 )

props은 수정해서는 안된다. 항상 동일한 입력값에 대해 동일한 결과를 반환하기 때문이다.

React Component

Posted on 2020-12-09

React Component

컴포넌트를 통해 UI를 재사용 가능한 개별적인 여러 조각으로 나누고, 각 조각을 개별적으로 살펴볼 수 있다.

주의할 점은 컴포넌트의 이름은 항상 대문자로 시작한다.

React는 소문자로 시작하는 컴포넌트를 DOM 태그로 처리합니다. 예를 들어 <div />는 HTML div 태그를 나타내지만, <Welcome />은 컴포넌트를 나타내며 범위 안에 Welcome이 있어야 합니다.

1. 함수 컴포넌트

JavaScript 함수이기 때문에 말 그대로 “함수 컴포넌트”라고 한다.

함수 컴포넌트는 재호출시 위의 FunctionComponent함수가 다시 호출된다.

컴포넌트를 만들고 render를 호출시 다음과 같은 과정이 일어난다.

  1. <FunctionComponent />, 엘리먼트로 ReactDOM.render()를 호출합니다.
  2. React는 {name: 'Sara'}를 props로 하여 FunctionComponent 컴포넌트를 호출합니다.
  3. FunctionComponent 컴포넌트는 결과적으로 <h1>함수 컴포넌트를 배우는 Sara</h1> 엘리먼트를 반환합니다.
  4. React DOM은 <h1>함수 컴포넌트를 배우는 Sara</h1> 엘리먼트와 일치하도록 DOM을 효율적으로 업데이트합니다.

컴포넌트 합성

컴포넌트는 자신의 출력에 다른 컴포넌트를 참조할 수 있다.

이는 모든 세부 단계에서 동일한 컴포넌트를 사용할 수 있음을 의미한다. React 앱에서는 버튼, 폼, 다이얼로그, 화면 등의 모든 것들이 컴포넌트로 표현된다.

App이라는 컴포넌트에서 Welcome 컴포넌트를 참조하였다.

함수에서 클래스 형태로 컴포넌트 변환하기

  1. React.Component를 확장하는 동일한 이름의 ES6 class를 생성합니다.
  2. render()라고 불리는 빈 메서드를 추가합니다.
  3. 함수의 내용을 render() 메서드 안으로 옮깁니다.
  4. render() 내용 안에 있는 props를 this.props로 변경합니다.
  5. 남아있는 빈 함수 선언을 삭제합니다.

FunctionComponent 컴포넌트는 이제 함수가 아닌 클래스로 정의된다.

render 메서드는 업데이트가 발생할 때마다 호출되지만, 같은 DOM 노드로 <FunctionComponent />을 렌더링하는 경우 FunctionComponent 클래스의 단일 인스턴스만 사용한다.

이것은 로컬 state와 생명주기 메서드와 같은 부가적인 기능을 사용할 수 있게 한다.

2. class 함수 형태

es6의 class 문법을 이용한 컴포넌트

클래스 컴포넌트는 재호출시 위의 Component 클래스 내부에서 render함수가 다시 호출된다.

클래스에 로컬 State 추가하기

  1. render() 메서드 안에 있는 this.props.name를 this.state.name로 변경합니다.
  2. 초기 this.state를 지정하는 class constructor (생성자)를 추가한다.

React 문법

JSX

위의 문법은 JSX라 하며 JavaScript를 확장한 문법이다. ( JavaScript Expand )

JSX는 React “엘리먼트(element)” 를 생성한다. 생성된 엘리먼트는 추후에 렌더링 된다.

JSX의 중괄호 안에는 유효한 모든 JavaScript 표현식을 넣을 수 있습니다.

  • 표현식 : 값으로 평가될 수 있는 문으로 표현식이 평가되면 새로운 값을 생성하거나 기존의 값을 참조한다.

    • 리터럴 표현식 - 인간이 이해할 수 있는 문자 또는 약속된 기호로 값을 생성하는 표기방식
    • 함수 호출문 표현식
    • 연산 표현식
    • 식별자 표현식

JSX 태그는 자식을 포함할 수 있습니다.

태그가 비어있다면 XML처럼 /> 를 이용해 바로 닫아주어야 합니다.

이는 닫는 태그가 없는 태그도 마찬가지다.

JSX 는 바로 사용할 수 없고 Babel을 이용해야 한다.

Babel은 JSX를 React.createElement() 호출로 컴파일한다.

React 엘리먼트를 루트 DOM 노드에 렌더링하려면 둘 다 ReactDOM.render함수에 인수로 전달한다.

위의 예제는 setInterval 함수를 통해서 1초마다 콜백함수 tick을 실행한다.

이는 콜백함수 tick 안의 ReactDOM.render 함수를 호출한다는 의미이다.

DOM

Posted on 2020-10-29

Dom

dom : documnet object model 의 약자로 문서의 계층적인 구조를 나타낸다.

요소 vs 노드

요소는 html 문서를 구성하는 태그를 나타낸다.

이러한 요소는 html문서가 파싱되는 과정에서 요소 노드 객체로 변한다.

html 태그(요소)는 요소노드, 어트리뷰트노드, 텍스트노드 등 3가지로 분리된다.

  • 요소노드(element node) : 태그(요소) 자체를 의미한다.
  • 어트리뷰트노드(attribute node) : html 태그의 속성이 변환된 노드로 위의 요소 노드와 형제 관계이다.
  • 텍스트노드(text node) : 태그의 텍스트로 요소 노드의 자식 요소로 할당된다.

노드는 위의 3가지 말고도 1가지를 더하여 총 4가지가 존재한다.

  • 문서 노드 : DOM 트리의 최상위에 존재하는 루트 노드로 Document 객체를 가르킨다.

    ​ 모든 노드는 Document 노드를 거쳐 찾을 수 밖에 없다.

위의 4가지(모든) 노드 객체는 Object, EventTarget, Node 인터페이스를 상속받는다.

  • Object : 객체
  • EventTarget : 이벤트를 발생시키는 객체
  • Node : 트리 구조를 구성하는 노드 객체

노드 객체는 각자 개별적으로 가진 기능이 있고 공통적으로 가지는 기능이 있다.

모든 노드 객체는 이벤트를 발생시킬 수 있어 EventTarget을 상속받는다.

요소 노드 취득

요소 노드 취득 관련 메서드는 HTMLCollection과 NodeList 반환 값으로 나눌 수 있다.

HTMLCollection는 살아 있는 객체라고 표현하는데 실시간으로 바뀐 요소 노드를 반영한다.

NodeList는 실시간으로 요소 노드를 반영하는 객체가 아닌, 과거의 정적인 상태를 유지한다.

그러나 childNode로 반환하는 객체는 NodeList이면서 살아 있는 객체이므로 주의해야한다.

getElementById 는 고유한 id값을 가진 요소노드만을 반환하고,

querySelector는 인수로 전달한 css 선택자 조건에 맞는 요소 노드 하나만을 반환한다.

getElementsByTagName , getElementsByClassName 등은 HTMLCollection 객체를 반환한다.

querySelectorAll 는 NodeList 객체를 반환한다.

노드 탐색

취득한 요소 노드를 기점으로 부모, 자식, 형제 노드를 찾는 것을 노드 탐색이라고 한다.

이때 어트리뷰트 노드는 요소 노드와 형제이기는 하나 요소 노드의 부모와 연결되어 있지는 않다.

때문에 Dom 트리 상에서는 형제관계가 반영되지 않으므로 주의해야 한다.

자식 노드 탐색

텍스트 노드 포함 탐색

  • childNodes
  • firstChild
  • lastChild

요소 노드만 탐색

  • children
  • firstElementChild
  • lastElementChild

자식 노드가 존재하는지 확인하려면 hasChildNodes 를 사용하면 되지만 텍스트 노드도 폼함하여 탐색하므로

요소노드만을 검색할 것이라면 children 을 사용해야 한다.

부모 노드 탐색

parentNode 를 사용하며 텍스트 노드는 자식 노드가 없는 Leaf Node, 즉 나뭇잎 같은 존재이므로 부모노드가 텍스트 노드인 경우는 없다.

형제 노드 탐색

어트리뷰트 노드는 요소 노드의 형제 노드이지만 부모 노드가 같은 형제 노드가 아니기 때문에 반환되지 않는다. 즉, 아래 프로퍼티는 텍스트 노드 또는 요소 노드만 반환한다.

여기서 텍스트 노드는 enter 키로 줄바꿈해서 생긴 공백 텍스트 노드를 의미한다.

텍스트 노드 포함 탐색

  • previousSibling
  • nextSibling

요소 노드만 탐색

  • previousElementSibling
  • nextElementSibling

insertAdjacentHTML : 기존 요소를 제거하지 않고 위치를 지정해 새로운 요소를 삽입

노드 이동

추가

  • appendChild : 마지막 노드 추가

  • insertBefore(newNode, childNode) : 첫 번째 인수로 전달받은 노드를 두 번째 인수로 전달받은 노드 앞에 추가

    복사

  • cloneNode([deep: true | false]) : 노드 복사 메서드로 true면 자손 노드를 포함해 복사하고 false(기본값) 라면 해당 노 드만 복사한다. 자손 노드를 복사하지 않으므로 텍스트 노드도 없다.

    교체

  • replaceChild(newChild, oldChild) : 자신을 호출한 노드의 자식 노드를 다른 노드로 교체한다.

    삭제

  • removeChild(child) : 인수로 전달한 노드를 DOM에서 삭제한다. 이때 인수로 전달한 노드는 removeChild 메서드를 호출한 노드의 자식 노드이어야 한다.

attributes ( 초기 값만 세팅 )

모든 HTML 요소에서 공통적으로 사용할 수 있지만 특정 HTML 요소에만 한정적으로 사용 가능한 어트리뷰트도 있다.

id, class, style 속은 모든 태그가 사용이 가능하지만, 위의 input 태그만이 type, value 등 독자적으로 가지는 속성도 존재한다.

HTML 문서가 파싱될 때 HTML 요소의 속성은 어트리뷰트 노드로 변환되어 요소 노드의 형제 노드로 추가된다.

이때 HTML 속성당 하나의 어트리뷰트 노드가 생성된다. 즉, 위 input 요소는 3개의 속성 있으므로 3개의 어트리뷰트 노드가 생성된다.

모든 어트리뷰트 노드의 참조는 유사 배열 객체이자 이터러블인 NamedNodeMap 객체에 담겨서 요소 노드의 attributes 프로퍼티에 저장된다.

attributes 프로퍼티는 읽기 전용 접근자 프로퍼티(getter) 이며 변경이 불가능하다.

getAttribute / setAttribute 를 사용하면 attributes를 통하지 않아도 속성 값을 얻거나 재할당할 수 있다.

HTML 문서가 파싱 -> HTML 요소의 속성당 어트리브튜 노드로 변환 -> 요소 노드의 attributes 프로퍼티에 유사 배열 객체이자 이터러블인 NamedNodeMap 객체에 담겨 저장

DOM 프로퍼티 ( 최신 속성값 상태 유지)

요소 노드 객체에는 HTML 어트리뷰트에 대응하는 프로퍼티가 존재하는데 이를 DOM 프로퍼티 라고 한다.

DOM 프로퍼티들은 HTML 어트리뷰트 값을 초기값으로 가지고 있다.

DOM 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티이므로 DOM 프로퍼티는 참조와 변경이 가능하다.

html 태그(요소)의 속성은 요소노드의 attribute 프로퍼티와 dom 프로퍼티는 중복으로 관리되는 것처럼 보인다.

  1. 요소 노드의 attributes 프로퍼티에서 관리하는 어트리뷰트 노드
  2. HTML 어트리뷰트에 대응하는 요소 노드의 프로퍼티(DOM 프로퍼티)

그러나 엄연히 다르다.

처음에 태그가 요소노드로 파싱되고 해당 태그의 속성이 어트리뷰트 노드로 변환되어 요소 노드의 attributes 프로퍼티에 저장된다.

이와는 별도로 태그의 속성 값은 요소 노드의 dom 프로퍼티에 할당된다.

요소 노드는 상태(state)를 가지고 있다.

attributes 노드에서 관리하는 어트리뷰트 값은 사용자의 입력에 의해 상태가 변경되어도 변하지 않고 HTML 어트리뷰트로 지정한 HTML 요소의 초기 상태를 그대로 유지한다.( 즉 초기 상태 값이다. )

반면에

사용자가 입력한 최신 상태는 HTML 어트리뷰트에 대응하는 요소 노드의 DOM 프로퍼티가 관리한다. DOM 프로퍼티는 사용자의 입력에 의한 상태 변화에 반응하여 언제나 최신 상태를 유지한다.

이처럼 attributes 프로퍼티는 HTML 요소의 초기 상태 값을 관리하고

DOM 프로퍼티는 사용자의 입력에 의해 변경되는 최신 상태를 관리한다.

html 속성에 대한 Dom 프로퍼티 대응

id, value(input) 등은 동일한 이름의 Dom 프로퍼티가 존재한다.

그러나 몇몇의 속성은 Dom 프로퍼티와 이름이 같지 않거나 아예 없다.

  • class 속성은 Dom 프로퍼티의 className, classList 로 대응한다.
  • for 어트리뷰트는 htmlFor 프로퍼티와 1:1 대응한다.
  • td 요소의 colspan 어트리뷰트는 대응하는 프로퍼티가 존재하지 않는다.

선택 정렬

  • 선택 정렬(selection sort)은 배열의 최소값을 검색하여 배열의 왼쪽부터 순차적으로 정렬을 반복하는 정렬 알고리즘이다.
  • 배열이 미정렬 상태이므로 최소값 검색에는 이진 검색이 아닌 선형 검색 알고리즘을 사용한다.
  • 선택 정렬은 버블 정렬보다 빠르다.
  • 시간 복잡도: O(n2)

삽입 정렬

  • 삽입 정렬(insertion sort)은 인덱스 1부터 왼쪽과 비교하면서 순차적으로 정렬을 반복하는 정렬 알고리즘이다.
  • 정렬이 진행됨에 따라 왼쪽에는 정렬이 종료된 값이 모이게 되고, 오른쪽에는 아직 정렬되지 않은 값이 남게 된다.
  • 선택 정렬은 최소값 검색이 필요하지만 삽입 정렬은 필요 없다.
  • 삽입 정렬은 평균 시나리오에서 선택 정렬과 유사하고(데이터 정렬 유형에 따라 차이가 있다) 버블 정렬보다 빠르다.
  • 시간 복잡도: O(n2)

this

this는 자신이 속한 객체 또는 자신이 생성할 인스턴스를 가리키는 자기 참조 변수(self-referencing variable)이다.

this를 통해 자신이 속한 객체 또는 자신이 생성할 인스턴스의 프로퍼티나 메서드를 참조할 수 있다.

this는 자바스크립트 엔진에 의해 암묵적으로 생성되며, 코드 어디서든 참조할 수 있다.

단, this가 가리키는 값, 즉 this 바인딩은 함수 호출 방식에 의해 동적으로 결정된다.

렉시컬 스코프와 this 바인딩은 결정 시기가 다르다.

함수의 상위 스코프를 결정하는 방식인 렉시컬 스코프(lexical scope)는 함수 정의가 평가되어 함수 객체가 생성되는 시점에 상위 스코프를 결정한다. 하지만 this 바인딩은 함수 호출 시점에 결정된다.

1) 일반 함수 호출 this

기본적으로 this는 전역객체가 할당된다.

만약에 strict mode가 적용된다면 일반 함수 내부의 this에는 undefined가 바인딩된다.

메서드 내에서 정의한 중첩 함수도 일반 함수로 호출되면 중첩 함수 내부의 this에는 전역 객체가 바인딩된다.

-> 일반 함수로 호출된 모든 함수(중첩 함수, 콜백 함수 포함) 내부의 this에는 전역 객체가 바인딩된다.

2) 메서드 호출 this

메서드 내부의 this에는 메서드를 호출한 객체, 즉 메서드를 호출할 때 메서드 이름 앞의 마침표(.) 연산자 앞에 기술한 객체가 바인딩된다. 주의할 것은 메서드 내부의 this는 메서드를 소유한 객체가 아닌 메서드를 호출한 객체에 바인딩된다는 것이다.

person 객체의 getName 프로퍼티가 가리키는 함수 객체는 person 객체에 포함된 것이 아니라 독립적으로 존재하는 별도의 객체다.

getName 프로퍼티가 가리키는 함수 객체, 즉 getName 메서드는 다른 객체의 프로퍼티에 할당하는 것으로 다른 객체의 메서드가 될 수도 있고 일반 변수에 할당하여 일반 함수로 호출될 수도 있다.

메서드 내부의 this는 프로퍼티로 메서드를 가리키고 있는 객체와는 관계가 없고 메서드를 호출한 객체에 바인딩된다.

3) 생성자 함수 this

생성자 함수 내부의 this에는 생성자 함수가 (미래에) 생성할 인스턴스가 바인딩된다.

생성자 함수는 이름 그대로 객체(인스턴스)를 생성하는 함수다. 일반 함수와 동일한 방법으로 생성자 함수를 정의하고 new 연산자와 함께 호출하면 해당 함수는 생성자 함수로 동작한다.

만약 new 연산자와 함께 생성자 함수를 호출하지 않으면 생성자 함수가 아니라 일반 함수로 동작한다.

apply/call/bind 메서드에 의한 간접 호출

Function.prototype.apply, Function.prototype.call 메서드는 this로 사용할 객체와 인수 리스트를 인수로 전달받아 함수를 호출

apply와 call 메서드의 본질적인 기능은 함수를 호출하는 것이다. apply와 call 메서드는 함수를 호출하면서 첫 번째 인수로 전달한 특정 객체를 호출한 함수의 this에 바인딩한다.

apply, call 은 호출할 함수에 인수를 전달하는 방식만 다를 뿐 동일하게 동작한다.

함수 호출 방식 this 바인딩
일반 함수 호출 전역 객체
메서드 호출 메서드를 호출한 객체
생성자 함수 호출 생성자 함수가 (미래에) 생성할 인스턴스
