요소 vs 노드

요소는 html 문서를 구성하는 태그를 나타낸다.

이러한 요소는 html문서가 파싱되는 과정에서 요소 노드 객체로 변한다.

html 태그(요소)는 요소노드, 어트리뷰트노드, 텍스트노드 등 3가지로 분리된다.

  • 요소노드(element node) : 태그(요소) 자체를 의미한다.
  • 어트리뷰트노드(attribute node) : html 태그의 속성이 변환된 노드로 위의 요소 노드와 형제 관계이다.
  • 텍스트노드(text node) : 태그의 텍스트로 요소 노드의 자식 요소로 할당된다.

노드는 위의 3가지 말고도 1가지를 더하여 총 4가지가 존재한다.

  • 문서 노드 : DOM 트리의 최상위에 존재하는 루트 노드로 Document 객체를 가르킨다.

    ​ 모든 노드는 Document 노드를 거쳐 찾을 수 밖에 없다.

위의 4가지(모든) 노드 객체는 Object, EventTarget, Node 인터페이스를 상속받는다.

  • Object : 객체
  • EventTarget : 이벤트를 발생시키는 객체
  • Node : 트리 구조를 구성하는 노드 객체

노드 객체는 각자 개별적으로 가진 기능이 있고 공통적으로 가지는 기능이 있다.

모든 노드 객체는 이벤트를 발생시킬 수 있어 EventTarget을 상속받는다.

요소 노드 취득

요소 노드 취득 관련 메서드는 HTMLCollection과 NodeList 반환 값으로 나눌 수 있다.

HTMLCollection는 살아 있는 객체라고 표현하는데 실시간으로 바뀐 요소 노드를 반영한다.

NodeList는 실시간으로 요소 노드를 반영하는 객체가 아닌, 과거의 정적인 상태를 유지한다.

그러나 childNode로 반환하는 객체는 NodeList이면서 살아 있는 객체이므로 주의해야한다.

getElementById 는 고유한 id값을 가진 요소노드만을 반환하고,

querySelector는 인수로 전달한 css 선택자 조건에 맞는 요소 노드 하나만을 반환한다.

getElementsByTagName , getElementsByClassName 등은 HTMLCollection 객체를 반환한다.

querySelectorAll 는 NodeList 객체를 반환한다.

노드 탐색

취득한 요소 노드를 기점으로 부모, 자식, 형제 노드를 찾는 것을 노드 탐색이라고 한다.

이때 어트리뷰트 노드는 요소 노드와 형제이기는 하나 요소 노드의 부모와 연결되어 있지는 않다.

때문에 Dom 트리 상에서는 형제관계가 반영되지 않으므로 주의해야 한다.

자식 노드 탐색

텍스트 노드 포함 탐색

  • childNodes
  • firstChild
  • lastChild

요소 노드만 탐색

  • children
  • firstElementChild
  • lastElementChild

자식 노드가 존재하는지 확인하려면 hasChildNodes 를 사용하면 되지만 텍스트 노드도 폼함하여 탐색하므로

요소노드만을 검색할 것이라면 children 을 사용해야 한다.

부모 노드 탐색

parentNode 를 사용하며 텍스트 노드는 자식 노드가 없는 Leaf Node, 즉 나뭇잎 같은 존재이므로 부모노드가 텍스트 노드인 경우는 없다.

형제 노드 탐색

어트리뷰트 노드는 요소 노드의 형제 노드이지만 부모 노드가 같은 형제 노드가 아니기 때문에 반환되지 않는다. 즉, 아래 프로퍼티는 텍스트 노드 또는 요소 노드만 반환한다.

여기서 텍스트 노드는 enter 키로 줄바꿈해서 생긴 공백 텍스트 노드를 의미한다.

텍스트 노드 포함 탐색

  • previousSibling
  • nextSibling

요소 노드만 탐색

  • previousElementSibling
  • nextElementSibling

insertAdjacentHTML : 기존 요소를 제거하지 않고 위치를 지정해 새로운 요소를 삽입

노드 이동

추가

  • appendChild : 마지막 노드 추가

  • insertBefore(newNode, childNode) : 첫 번째 인수로 전달받은 노드를 두 번째 인수로 전달받은 노드 앞에 추가

    복사

  • cloneNode([deep: true | false]) : 노드 복사 메서드로 true면 자손 노드를 포함해 복사하고 false(기본값) 라면 해당 노 드만 복사한다. 자손 노드를 복사하지 않으므로 텍스트 노드도 없다.

    교체

  • replaceChild(newChild, oldChild) : 자신을 호출한 노드의 자식 노드를 다른 노드로 교체한다.

    삭제

  • removeChild(child) : 인수로 전달한 노드를 DOM에서 삭제한다. 이때 인수로 전달한 노드는 removeChild 메서드를 호출한 노드의 자식 노드이어야 한다.

attributes ( 초기 값만 세팅 )

모든 HTML 요소에서 공통적으로 사용할 수 있지만 특정 HTML 요소에만 한정적으로 사용 가능한 어트리뷰트도 있다.

id, class, style 속은 모든 태그가 사용이 가능하지만, 위의 input 태그만이 type, value 등 독자적으로 가지는 속성도 존재한다.

HTML 문서가 파싱될 때 HTML 요소의 속성은 어트리뷰트 노드로 변환되어 요소 노드의 형제 노드로 추가된다.

이때 HTML 속성당 하나의 어트리뷰트 노드가 생성된다. 즉, 위 input 요소는 3개의 속성 있으므로 3개의 어트리뷰트 노드가 생성된다.

모든 어트리뷰트 노드의 참조는 유사 배열 객체이자 이터러블인 NamedNodeMap 객체에 담겨서 요소 노드의 attributes 프로퍼티에 저장된다.

attributes 프로퍼티는 읽기 전용 접근자 프로퍼티(getter) 이며 변경이 불가능하다.

getAttribute / setAttribute 를 사용하면 attributes를 통하지 않아도 속성 값을 얻거나 재할당할 수 있다.

HTML 문서가 파싱 -> HTML 요소의 속성당 어트리브튜 노드로 변환 -> 요소 노드의 attributes 프로퍼티에 유사 배열 객체이자 이터러블인 NamedNodeMap 객체에 담겨 저장

DOM 프로퍼티 ( 최신 속성값 상태 유지)

요소 노드 객체에는 HTML 어트리뷰트에 대응하는 프로퍼티가 존재하는데 이를 DOM 프로퍼티 라고 한다.

DOM 프로퍼티들은 HTML 어트리뷰트 값을 초기값으로 가지고 있다.

DOM 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티이므로 DOM 프로퍼티는 참조와 변경이 가능하다.

html 태그(요소)의 속성은 요소노드의 attribute 프로퍼티와 dom 프로퍼티는 중복으로 관리되는 것처럼 보인다.

  1. 요소 노드의 attributes 프로퍼티에서 관리하는 어트리뷰트 노드
  2. HTML 어트리뷰트에 대응하는 요소 노드의 프로퍼티(DOM 프로퍼티)

그러나 엄연히 다르다.

처음에 태그가 요소노드로 파싱되고 해당 태그의 속성이 어트리뷰트 노드로 변환되어 요소 노드의 attributes 프로퍼티에 저장된다.

이와는 별도로 태그의 속성 값은 요소 노드의 dom 프로퍼티에 할당된다.

요소 노드는 상태(state)를 가지고 있다.

attributes 노드에서 관리하는 어트리뷰트 값은 사용자의 입력에 의해 상태가 변경되어도 변하지 않고 HTML 어트리뷰트로 지정한 HTML 요소의 초기 상태를 그대로 유지한다.( 즉 초기 상태 값이다. )

반면에

사용자가 입력한 최신 상태는 HTML 어트리뷰트에 대응하는 요소 노드의 DOM 프로퍼티가 관리한다. DOM 프로퍼티는 사용자의 입력에 의한 상태 변화에 반응하여 언제나 최신 상태를 유지한다.

이처럼 attributes 프로퍼티는 HTML 요소의 초기 상태 값을 관리하고

DOM 프로퍼티는 사용자의 입력에 의해 변경되는 최신 상태를 관리한다.

html 속성에 대한 Dom 프로퍼티 대응

id, value(input) 등은 동일한 이름의 Dom 프로퍼티가 존재한다.

그러나 몇몇의 속성은 Dom 프로퍼티와 이름이 같지 않거나 아예 없다.

  • class 속성은 Dom 프로퍼티의 className, classList 로 대응한다.
  • for 어트리뷰트는 htmlFor 프로퍼티와 1:1 대응한다.
  • td 요소의 colspan 어트리뷰트는 대응하는 프로퍼티가 존재하지 않는다.

Homework5

선택 정렬을 통해 주어진 배열(array)을 정렬하는 함수를 구현하라. 단, 어떠한 빌트인 함수도 사용하지 않고 for 문을 사용하여 구현하여야 한다.

선택 정렬

  • 선택 정렬(selection sort)은 배열의 최소값을 검색하여 배열의 왼쪽부터 순차적으로 정렬을 반복하는 정렬 알고리즘이다.
  • 배열이 미정렬 상태이므로 최소값 검색에는 이진 검색이 아닌 선형 검색 알고리즘을 사용한다.
  • 선택 정렬은 버블 정렬보다 빠르다.
  • 시간 복잡도: O(n2)

삽입 정렬

  • 삽입 정렬(insertion sort)은 인덱스 1부터 왼쪽과 비교하면서 순차적으로 정렬을 반복하는 정렬 알고리즘이다.
  • 정렬이 진행됨에 따라 왼쪽에는 정렬이 종료된 값이 모이게 되고, 오른쪽에는 아직 정렬되지 않은 값이 남게 된다.
  • 선택 정렬은 최소값 검색이 필요하지만 삽입 정렬은 필요 없다.
  • 삽입 정렬은 평균 시나리오에서 선택 정렬과 유사하고(데이터 정렬 유형에 따라 차이가 있다) 버블 정렬보다 빠르다.
  • 시간 복잡도: O(n2)

this

this는 자신이 속한 객체 또는 자신이 생성할 인스턴스를 가리키는 자기 참조 변수(self-referencing variable)이다.

this를 통해 자신이 속한 객체 또는 자신이 생성할 인스턴스의 프로퍼티나 메서드를 참조할 수 있다.

this는 자바스크립트 엔진에 의해 암묵적으로 생성되며, 코드 어디서든 참조할 수 있다.

단, this가 가리키는 값, 즉 this 바인딩은 함수 호출 방식에 의해 동적으로 결정된다.

렉시컬 스코프와 this 바인딩은 결정 시기가 다르다.

함수의 상위 스코프를 결정하는 방식인 렉시컬 스코프(lexical scope)는 함수 정의가 평가되어 함수 객체가 생성되는 시점에 상위 스코프를 결정한다. 하지만 this 바인딩은 함수 호출 시점에 결정된다.

1) 일반 함수 호출 this

기본적으로 this는 전역객체가 할당된다.

만약에 strict mode가 적용된다면 일반 함수 내부의 this에는 undefined가 바인딩된다.

메서드 내에서 정의한 중첩 함수도 일반 함수로 호출되면 중첩 함수 내부의 this에는 전역 객체가 바인딩된다.

-> 일반 함수로 호출된 모든 함수(중첩 함수, 콜백 함수 포함) 내부의 this에는 전역 객체가 바인딩된다.

2) 메서드 호출 this

메서드 내부의 this에는 메서드를 호출한 객체, 즉 메서드를 호출할 때 메서드 이름 앞의 마침표(.) 연산자 앞에 기술한 객체가 바인딩된다. 주의할 것은 메서드 내부의 this는 메서드를 소유한 객체가 아닌 메서드를 호출한 객체에 바인딩된다는 것이다.

person 객체의 getName 프로퍼티가 가리키는 함수 객체는 person 객체에 포함된 것이 아니라 독립적으로 존재하는 별도의 객체다.

getName 프로퍼티가 가리키는 함수 객체, 즉 getName 메서드는 다른 객체의 프로퍼티에 할당하는 것으로 다른 객체의 메서드가 될 수도 있고 일반 변수에 할당하여 일반 함수로 호출될 수도 있다.

메서드 내부의 this는 프로퍼티로 메서드를 가리키고 있는 객체와는 관계가 없고 메서드를 호출한 객체에 바인딩된다.

3) 생성자 함수 this

생성자 함수 내부의 this에는 생성자 함수가 (미래에) 생성할 인스턴스가 바인딩된다.

생성자 함수는 이름 그대로 객체(인스턴스)를 생성하는 함수다. 일반 함수와 동일한 방법으로 생성자 함수를 정의하고 new 연산자와 함께 호출하면 해당 함수는 생성자 함수로 동작한다.

만약 new 연산자와 함께 생성자 함수를 호출하지 않으면 생성자 함수가 아니라 일반 함수로 동작한다.

apply/call/bind 메서드에 의한 간접 호출

Function.prototype.apply, Function.prototype.call 메서드는 this로 사용할 객체와 인수 리스트를 인수로 전달받아 함수를 호출

apply와 call 메서드의 본질적인 기능은 함수를 호출하는 것이다. apply와 call 메서드는 함수를 호출하면서 첫 번째 인수로 전달한 특정 객체를 호출한 함수의 this에 바인딩한다.

apply, call 은 호출할 함수에 인수를 전달하는 방식만 다를 뿐 동일하게 동작한다.

함수 호출 방식 this 바인딩
일반 함수 호출 전역 객체
메서드 호출 메서드를 호출한 객체
생성자 함수 호출 생성자 함수가 (미래에) 생성할 인스턴스
Function.prototype.apply/call/bind 메서드에 의한 간접 호출 Function.prototype.apply/call/bind 메서드에 첫 번째 인수로 전달한 객체

Built in Object ( 빌트 인 객체 )

JS 객체의 정류

  1. 표준 빌트인 객체 : ECMAScript 사양에 정의된 객체로서, 애플리케이션 전역의 공통 기능을 제공

​ 빌트인 객체는 ECMAScript 사양에 정의된 객체이므로 자바스크립트 실행 환경(브라우저 또는 Node.js 환경)과 관계없이 언제나 사용할 수 있다. 표준 빌트인 객 체는 전역 객체의 프로퍼티로서 제공된다. 따라서 별도의 선언 없이 전역 변수처럼 언제나 참조할 수 있다.

  1. 호스트 객체 : ECMAScript 사양에 정의되어 있지 않지만 자바스크립트 실행 환경(브라우저 환경 또는 Node.js 환경 등에서 에서 추가로 제공하는 객체

  2. 사용자 정의 객체 : 사용자가 직접 정의한 객체

표준 빌트인 객체

자바스크립트는 Object, String, Number, Boolean, Symbol, Date, Math, RegExp, Array, Map/Set, WeakMap/WeakSet, Function, Promise, Reflect, Proxy, JSON, Error 등의 빌트인 객체를 제공한다.

Math, Reflect, JSON을 제외한 표준 빌트인 객체는 모두 인스턴스를 생성할 수 있는 생성자 함수 객체다. 생성자 함수 객체인 표준 빌트인 객체는 프로토타입 메서드와 정적 메소드를 제공하고 생성자 함수 객체가 아닌 표준 빌트인 객체는 정적 메소드만 제공한다.

예를 들어, 표준 빌트인 객체인 String, Number, Boolean, Function, Array, Date는 생성자 함수로 호출하여 인스턴스를 생성할 수 있다.

생성자 함수인 표준 빌트인 객체가 생성한 인스턴스의 프로토타입은 표준 빌트인 객체의 prototype 프로퍼티에 바인딩된 객체이다.

원시값 / 래퍼 객체

문자열이나 숫자, 불리언 등의 원시값이 있는데도 문자열, 숫자, 불리언 객체를 생성하는 String, Number, Boolean 등의 표준 빌트인 생성자 함수가 존재한다.
생성자 함수로서 함수 객체를 호출할 때는 new 키워드를 호출문 앞에 붙이는 것으로서 생성자 함수로서 내부 메서드 [[Construct]] 가 호출된다.

만약에 123이라는 숫자값을 각각 String, Number, Boolean 등의 함수 호출문에 인수로 전달하고 new 키워드로 생성사 함수로서 호출하면 [‘123’], [123], [true] 의 객체가 호출된다.

보통은 저렇게 원시값을 굳이 객체로 바꾸지는 않는다.
그러나 이러한 방식은 명시적인 사용하지는 않지만, 자바스크립트 엔진에 의한 내부적으로 동작이되는 때가 있으며 반드시 필요한 암묵적인 기능이다.

위의 예시에서 원시값은 객체가 아니므로 프로퍼티나 메서드를 가질 수 없는데도 원시값인 문자열이 마치 객체처럼 동작한다.

원시값인 문자열, 숫자, 불리언 값의 경우 이들 원시값에 대해 마치 객체처럼 마침표 표기법(또는 대괄호 표기법)으로 접근하면, 자바스크립트 엔진이 일시적으로 원시값을 연관된 객체로 변환해 주기 때문이다.

원시값을 객체처럼 사용하면 자바스크립트 엔진은 암묵적으로 연관된 객체를 생성하여 생성된 객체로 프로퍼티에 접근하거나 메서드를 호출하고 다시 원시값으로 되돌린다.

다시 정리하자면 우리가 문자열에 대해 객체처럼 마침표 표기법으로 함수를 사용하는 것은 내부적으로 임시객체를 호출하고 함수 실행이 끝나면
다시 원래 원시값으로 되돌리는 것이다.

문자열, 숫자, 불리언 값등 원시값에 대해 객체처럼 접근하면 생성되는 임시 객체를 래퍼 객체라고 한다.

원시값에 마침표 표기법 사용 -> 일시적으로 해당 원시값에 연관된 객체로 변호나 -> 연관 객체로 프로퍼티/ 메소드에 접근 호출하고 다시 원시값으로 되돌림

문자열 래퍼 객체인 String 생성자 함수의 인스턴스는 String.prototype의 메서드를 상속받아 사용할 수 있다.

래퍼 객체의 처리가 종료되면 래퍼 객체의 [[StringData]] 내부 슬롯에 할당된 원시값으로 원래의 상태, 즉 식별자가 원시값을 갖도록 되돌리고 래퍼 객체는 가비지 컬렉션의 대상이 된다.
그렇게 되면 임시적으로 호출된 객체는 가비지 컬렉션의 대상이 되므로 메모리에서 해제가된다.

이는 숫자 타입 값이나 불리언 타입 값도 마찬가지다.

그리고 문자열, 숫자, 불리언, 심벌 이외의 원시값, 즉 null과 undefined는 래퍼 객체를 생성하지 않는다. 따라서 null과 undefined 값을 객체처럼 사용하면 에러가 발생한다.

전역객체

코드가 실행되기 이전 단계에 자바스크립트 엔진에 의해 어떤 객체보다도 먼저 생성되는 특수한 객체이며, 어떤 객체에도 속하지 않은 최상위 객체다.

브라우저 환경에서는 window(또는 self, this, frames)가 전역 객체를 가리키지만 Node.js 환경에서는 global이 전역 객체를 가리킨다.

전역 객체는 표준 빌트인 객체(Object, String, Number, Function, Array 등)와 환경에 따른 호스트 객체(클라이언트 web API 또는 Node.js의 호스트 API), 그리고 var 키워드로 선언한 전역 변수와 전역 함수를 프로퍼티로 갖는다.

var키워드 외에 let이나 const 키워드로 선언한 전역 변수는 전역 객체의 프로퍼티가 아니다.

전역객체는 다음과 같은 특징을 가진다.

  • 전역 객체는 개발자가 의도적으로 생성할 수 없다. 즉, 전역 객체를 생성할 수 있는 생성자 함수가 제공되지 않는다.

  • 전역 객체의 프로퍼티를 참조할 때 window(또는 global)를 생략할 수 있다.

strict mod

자바스크립트는 사용자가 실수로 작성한 코드를 암묵적으로 변환하여 오류의 발생을 줄인다. 얼핏보면 장점으로 보이나 장기적인 관점에서 봤을 때 의도치 않은 에러가 발생할 수 있다.

이를 방지하기 위해 strict mod를 사용한다.

혹 외부 라이브러리를 사용한다면 저역 환경에 strict mod를 사용하다간 충돌이 발생할 수 있으므로 즉시 실행함수 (function(){}()) 내부에 사용하는 것이 좋다.

strict 를 사용한 후 변화

  1. 선언하지 않은 변수를 참조하면 에러가 발생한다.

원래 자바스크립트 엔진은 위의 상황에서 암묵적으로 전역 객체에 x 프로퍼티를 동적 생성한다.

strict 모드를 사용함으로서 선언하지 않은 변수 1이란 값을 할당한 상황에서 정의되지 않은 변수라고 에러를 발생시킨다.

  1. delete 연산자

delete 연산자로 변수, 함수, 매개변수를 삭제하면 SyntaxError가 발생한다.

  1. 매개변수 이름의 중복

중복된 매개변수 이름을 사용하면 SyntaxError가 발생한다.

  1. 일반 함수의 this

strict mode 에서 함수를 일반 함수로서 호출하면 this에 undefined가 바인딩된다.

원래 대로라면 window라는 전역 객체를 바인딩 한다.

  1. arguments 객체

매개변수에 전달된 인수를 재할당하여 변경하여도 arguments 객체에 반영되지 않는다.

Prototype

객체 지향 언어 : 객체(object)의 집합으로 프로그램을 표현하려는 프로그래밍 언어

자바스크립트는 프로토타입 기반의 객체지향 프로그래밍 언어 이다.

자바스크립트는 객체 기반의 프로그래밍 언어이며 자바스크립트를 이루고 있는 “거의 모든 것”이 객체다. 원시 타입(primitive type)의 값을 제외한 나머지 값들(함수, 배열, 정규표현식 등)은 모두 객체다.

원시 타입

  • string
  • number
  • boolean
  • null
  • undefined
  • symbol

객체 타입

  • 객체
    • 배열
    • 함수
    • 정규표현식

사람에게는 다양한 속성이 있으나 우리가 구현하려는 프로그램에서는 사람의 “이름”과 “주소”라는 속성에만 관심이 있다고 가정하자. 이처럼 다양한 속성 중에서 프로그램에 필요한 속성만 간추려 내어 표현하는 것을 **추상화(abstraction)**라 한다.

이처럼 객체지향 프로그래밍은 객체의 **상태(state)**를 나타내는 데이터와 상태 데이터를 조작할 수 있는 **동작(behavior)**을 하나의 논리적인 단위로 묶어 생각한다.

객체 : 상태 데이터와 동작을 하나의 논리적인 단위로 묶은 복합적인 자료구조

자바스크립트의 객체에서는 상태 데이터를 프로퍼티(property), 동작을 메서드(method)라 부른다.

상속과 프로토타입

어떤 객체의 프로퍼티 또는 메서드를 다른 객체가 상속받아 그대로 사용할 수 있는 것을 말한다.

상속을 사용하면 불필요한 중복을 제거할 수 있고, 이는 기존의 코드를 재사용하는 것으로 구현이 가능하다.

생성자 함수를 이용하면 미리 구현 템플릿으로 형태가 같은 객체(인스턴스)를 여러개로 생성이 가능하다.

그런데 위의 예제에서는 getArea라는 함수는 중복적인 내용이다.

중복적인 내용을 각 인스턴스가 소유하는 것보다는 하나만 생성하여 이를 공유하는 것이 메모리상으로 효율적이다.

모든 생성자 함수는 해당 함수와 짝이 되는 프로토 타입을 가지며, 생성된 객체는 해당 프로토타입의 모든 프로퍼티와 메서드를 상속받는다.

프로토타입

프로토타입 객체(또는 줄여서 프로토타입)란 객체지향 프로그래밍의 근간을 이루는 객체 간 상속을 구현하기 위해 사용된다.

프로토타입은 어떤 객체의 상위(부모) 객체의 역할을 하는 객체로서 다른 객체에 공유 프로퍼티(메서드 포함)를 제공한다. 프로토타입을 상속받은 하위(자식) 객체는 상위 객체의 프로퍼티를 자신의 프로퍼티처럼 자유롭게 사용할 수 있다.

모든 객체는 하나의 프로토타입을 갖는다.([[Prototype]]) 그리고 모든 프로토타입은 생성자 함수와 연결되어 있다.

객체 리터럴에 의해 생성된 객체의 프로토타입은 Object.prototype이고,

생성자 함수에 의해 생성된 객체의 프로토타입은 생성자 함수의 prototype 프로퍼티에 바인딩되어 있는 객체다.

[[Prototype]] 내부 슬롯에는 직접 접근할 수 없지만, proto 접근자 프로퍼티를 통해 자신의 프로토타입, 즉 자신의 [[Prototype]] 내부 슬롯이 가리키는 프로토타입에 간접적으로 접근할 수 있다.

그리고 프로토타입은 자신의 constructor 프로퍼티를 통해 생성자 함수에 접근할 수 있고, 생성자 함수는 자신의 prototype 프로퍼티를 통해 프로토타입에 접근할 수 있다.

proto

모든 객체는 proto 접근자 프로퍼티를 통해 자신의 프로토타입, 즉 [[Prototype]] 내부 슬롯에 간접적으로 접근할 수 있다.

proto 접근자 프로퍼티는 상속을 통해 사용된다.

proto 접근자 프로퍼티는 객체가 직접 소유하는 프로퍼티가 아니라 Object.prototype의 프로퍼티다. 모든 객체는 상속을 통해 Object.prototype.proto 접근자 프로퍼티를 사용할 수 있다.

Object.prototype

모든 객체는 프로토타입의 계층 구조인 프로토타입 체인에 묶여 있다. 자바스크립트 엔진은 객체의 프로퍼티(메서드 포함)에 접근하려고 할 때 해당 객체에 접근하려는 프로퍼티가 없다면 proto 접근자 프로퍼티가 가리키는 참조를 따라 자신의 부모 역할을 하는 프로토타입의 프로퍼티를 순차적으로 검색한다.

프로토타입 체인의 종점, 즉 프로토타입 체인의 최상위 객체는 Object.prototype이며, 이 객체의 프로퍼티와 메서드는 모든 객체에게 상속된다.

Object.create() 함수를 사용하면 해당 객체의 prototype을 지정할 수 있는데 저렇게 명시적으로 지정을 하면 프로토타입으로부터 상속을 받을 수 없다.

생성자 함수 prototype

함수 객체만이 소유하는 prototype 프로퍼티는 생성자 함수가 생성할 인스턴스의 프로토타입을 가리킨다.

일반 객체는 프로토 타입을 Object.prototype을 가지며,

생성사 함수에 의함 객체는 프로토 타입을 생성자 함수의 prototype로 가진다.

생정자 함수의 prototype은 프로토 타입으로 Object.prototype을 가진다.

Object.prototype -> 생성자 함수 prototype -> 인스턴스

Object.prototype -> 일반 객체

일급객체

  • 무명의 리터럴로 생성할 수 있다. 즉, 런타임에 생성이 가능하다.
  • 변수나 자료구조(객체, 배열 등)에 저장할 수 있다.
  • 함수의 매개변수에게 전달할 수 있다.
  • 함수의 반환값으로 사용할 수 있다.

위의 조건을 만족하면 일급 객체라 하는데

자바스크립트 함수 객체의 위의 조건을 모두 충족하므로 일극 객체라 할 수 있다.

함수는 객체이나 일반 객체와는 차이가 있다.

일반 객체는 호출할 수 없지만 함수 객체는 호출할 수 있다. 그리고 함수 객체는 일반 객체에는 없는 함수 고유의 프로퍼티를 소유한다.

자바스크립트 에서 함수객체는 _ proto _ 프로퍼티 외에도

  • arguments

  • caller

  • length

  • name

  • prototype

    등 일반 객체에는 없는 5개의 추가 프로퍼티 또한 가지고 있다.

일반 객체는 _ proto _ 만 가지고 있다.

arguments 프로퍼티

함수 객체의 arguments 프로퍼티 값은 arguments 객체다.

arguments 객체는 함수 호출 시 전달된 인수(argument)들의 정보를 담고 있는 순회 가능한 유사 배열 객체이며, 함수 내부에서 지역 변수처럼 사용된다.

함수가 호출되면 함수 몸체 내에서 암묵적으로 매개변수가 선언되고 undefined로 초기화된 이후 인수가 할당된다.

모든 인수는 암묵적으로 arguments 객체의 프로퍼티로 보관된다.

유사 배열 객체는 배열이 아니므로 배열 메서드를 사용할 경우 에러가 발생한다.

일반 객체처럼 사용하려면

  • Function.prototype.call

  • Function.prototype.apply

등으로 유사 배열객체를 배열로 변환해 줘야 한다.

caller 프로퍼티

함수 자신을 호출한 함수를 가르킨다. 만약에 호출한 함수가 자기자신이면 null 값이 할당되고

자신이 매개변수에 할당된 콜백함수로서 해당 고차함수에 의해 호출된다면 그 고차함수가 값으로 할당된다.

length 프로퍼티

함수를 정의할 때 선언한 매개변수의 개수가 값으로 할당된다.

위의 arguments 프로퍼티는 객체값으로서 length 프로퍼티 값을 사용해서 나온 숫자값과 와 헷갈릴 수 있으므로 차이점을 정리해본다.

length 프로퍼티는 생성할 때 작성한 매개변수의 개수이고,

arguments 프로퍼티의 length는 호출할 때 전달한 인수의 개수를 출력한다.

name 프로퍼티

es5에서는 익명 함수 표현식일 경우 빈 문자열을 값으로 가지지만,

es6부터는 함수객체를 가르키는 식별자를 가르킨다.

요컨대 함수선언문에서 함수 이름과 똑같은 암묵적으로 생성된 식별자, 일반 변수로서 사용되어 함수 리터럴을 할당받은 식별자등이 값이 된다는 말이다.

_ proto _ 접근자 프로퍼티

모든 객체는 [[Prototype]]이라는 내부 슬롯을 갖는다. [[Prototype]] 내부 슬롯은 객체 지향 프로그래밍의 상속을 구현하는 프로토타입 객체를 가리킨다.

proto 는 [[Prototype]] 내부 슬롯이 가리키는 프로토타입 객체에 접근하기 위해 사용하는 접근자 프로퍼티다.

prototype 프로퍼티

prototype 프로퍼티는 생성자 함수만이 소유하는 프로퍼티다. 일반 객체와 생성자 함수로 호출할 수 없는 함수에는 prototype 프로퍼티가 없다.

prototype 프로퍼티는 함수가 객체를 생성하는 생성자 함수로 호출될 때 생성자 함수가 생성할 인스턴스의 프로토타입 객체를 가리킨다.
